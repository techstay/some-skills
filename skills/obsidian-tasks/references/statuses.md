# Tasks 状态系统参考

## 内置状态

Tasks 默认识别两种核心状态（无需额外配置）：

| 状态符号 | Markdown | 状态类型 | 含义 |
|----------|----------|----------|------|
| `[ ]` | `- [ ] 任务` | `TODO` | 未完成 |
| `[x]` 或 `[X]` | `- [x] 任务` | `DONE` | 已完成 |

## 状态类型 (Status Types)

每个状态都有一个**类型**，控制其行为：

| 类型 | 含义 | `done`/`not done` 匹配 |
|------|------|----------------------|
| `TODO` | 待办 | → `not done` |
| `IN_PROGRESS` | 进行中 | → `not done` |
| `ON_HOLD` | 暂停/搁置 | → `not done` |
| `DONE` | 已完成 | → `done` |
| `CANCELLED` | 已取消 | → `done` |
| `NON_TASK` | 非任务（如笔记行） | → `done` |

## 自定义状态

可通过插件设置定义额外状态，使用任意单个字符作为状态符号：

```markdown
- [/] 正在进行中
- [-] 已取消
- [?] 待确认
- [!] 重要待办
- [<] 推迟
```

每个自定义状态需配置：
- **状态符号**：`[` 和 `]` 之间的字符
- **状态名称**：如 "In Progress"
- **下一个状态符号**：点击复选框后切换到的状态
- **状态类型**：从上述 6 种类型中选择

### 自定义状态与重复任务的交互

- 状态类型**变为** `DONE` 时 → 添加完成日期 + 触发重复任务创建新实例
- 状态类型**从** `DONE` 变为其他时 → 移除完成日期

### CSS 样式

自定义状态（非 `[ ]` 和 `[x]`）需要通过 CSS 片段或主题才能在预览和查询中正确显示复选框外观。

## 状态相关查询

| 指令 | 含义 |
|------|------|
| `status.name includes "In Progress"` | 状态名称包含文本 |
| `status.name does not include "Done"` | 状态名称不包含 |
| `status.type is TODO` | 状态类型为 TODO |
| `status.type is not DONE` | 状态类型不是 DONE |
| `sort by status` | 按状态排序 |
| `group by status` | 按状态分组 |

## AI 操作任务的规则

当 AI 帮用户编辑 `.md` 文件中的任务时：
- 完成任务：将 `[ ]` 改为 `[x]`，并追加 ` ✅ YYYY-MM-DD`
- 取消任务：将状态改为 `[-]`，并追加 ` ❌ YYYY-MM-DD`
- 设置进行中：将 `[ ]` 改为 `[/]`
- **未知状态符号默认视为 TODO 类型**
