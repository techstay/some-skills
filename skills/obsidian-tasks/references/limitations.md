# Tasks 限制与注意事项

## 核心限制

### 单行任务
任务**必须是单行**清单项。多行任务中第一行之后的内容会被忽略（原文件中的内容不受影响，只是 Tasks 不处理）。

### 日期格式
- 仅支持 `YYYY-MM-DD` 格式
- 不支持时间（无 HH:MM）
- 不可将日期格式化为 Wikilink（如 `[[2026-04-01]]`）

### 元数据解析顺序
Tasks 从行尾**向前**解析元数据。一旦遇到无法识别的内容，解析停止。
- **标签和块链接必须放在日期、优先级等元数据之后**
- 错误示例：`- [ ] 任务 #标签 📅 2026-04-01`（标签在日期前面，日期可能不会被解析）

## 不支持的场景

| 场景 | 是否支持 |
|------|----------|
| Canvas 卡片中的任务 | ❌ 不读取 |
| 代码块中的任务（如 Admonitions 插件） | ❌ 不读取 |
| HTML 注释 `<!-- -->` 中的任务 | ❌ 不读取 |
| Markdown 注释 `%%` 中的任务 | ❌ 不读取 |
| 块引用中的任务 | ⚠️ 可读取，但 Live Preview 下点击复选框可能行为异常 |
| 标注（Callout）中的任务 | ⚠️ 可读取，但 Live Preview 下可能不正常 |
| 常规脚注 `[^1]` 中的任务 | ❌ 不支持 |
| 行内脚注 `^[text]` 中的任务 | ✅ 支持 |
| 非 `.md` 文件 | ❌ 仅支持 `.md` 文件 |

## 重复任务限制

- 必须搭配至少一个日期（due/scheduled/start），否则重复规则无效
- 不建议在每日笔记（Daily Note）中创建重复任务
- 完成后新实例的 `id` 和 `dependsOn` 字段会被清除

## 有序列表中的任务

Tasks 可以读取有序列表中的任务（`1. [ ]`、`2) [ ]`），但查询结果中会显示为普通项目符号列表项。

## 全局过滤器

如果启用了全局过滤器（如 `#task`），则该标签会在读取时从任务行中**移除**。不要在查询中使用全局过滤器对应的标签。

## 无效日期

无效日期（如 `📅 2026-02-30`）的任务会在基于日期的查询中被**静默忽略**。可通过以下查询找出无效日期任务：

```
filter by function !task.due.isValid || !task.scheduled.isValid || !task.start.isValid
```

## 不在插件范围内的功能

以下功能 Tasks 插件不提供：
- 时间追踪 / 时间统计
- 生产力统计 / 番茄工作法
- 通知 / 提醒推送
- 看板视图（Kanban）
