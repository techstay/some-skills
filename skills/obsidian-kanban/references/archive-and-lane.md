# 归档系统与列设置

## 归档系统

### 归档数据存储

- 归档数据存储在同一个 `.md` 文件的 `%% kanban:archive %%` JSON 注释块中
- 归档数据与看板数据在同一文件内，便于版本控制和同步
- **不要手动编辑此注释块**，除非你知道自己在做什么

### 归档触发方式

| 方式 | 操作 |
|------|------|
| 命令面板归档全部 | `Ctrl+P` → `Kanban: Archive completed cards` |
| 手动归档单个卡片 | 右键卡片 → Archive |
| 按列归档 | 列右上角菜单 → Archive completed cards |
| 自动归档 | 勾选卡片后自动归档（需在设置中配置） |

### 查看和管理归档

- 通过看板顶部的 **归档按钮**（📦 图标）查看所有归档条目
- 已归档卡片可 **恢复（Restore）** 回原列
- 归档条目按归档时间倒序排列

### 归档配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `max-archive-size` | `-1`（无限） | 归档保留条数上限，超出后自动清理最旧的条目 |
| `archive-with-date` | `true` | 归档时自动附加完成日期和时间戳 |
| `archive-date-time-format` | `"YYYY-MM-DD HH:mm"` | 归档日期的显示格式（moment.js 语法） |
| `archive-date-time-separator` | `" "` | 归档日期和时间之间的分隔符 |

### 归档最佳实践

- 定期清理归档，避免文件过大影响性能
- 设置 `max-archive-size` 为合理值（如 50-200）
- 重要完成的卡片建议先创建笔记再归档，保留完整记录

## 列设置（Lane Settings）

每列右上角菜单（⋮ 图标）中可独立配置，**覆盖全局设置**：

### 排序方式

| 选项 | 说明 |
|------|------|
| **不排序** | 手动拖拽排序（默认） |
| **按日期排序** | 按卡片日期升序/降序排列 |
| **按文字排序** | 按卡片标题字母/拼音顺序排列 |

### 新卡片插入位置

| 选项 | 说明 |
|------|------|
| **顶部插入** | 新卡片出现在列的最上方 |
| **底部插入** | 新卡片出现在列的最下方（默认） |

> 此设置可覆盖全局的 `new-card-insertion-method`，允许不同列使用不同的插入策略。

### 列操作

| 操作 | 说明 |
|------|------|
| **归档已完成卡片** | 一键归档本列所有已勾选（`- [x]`）的卡片 |
| **折叠/展开** | 折叠本列内容（需全局开启 `list-collapse`） |
| **删除列** | 删除整列及其所有卡片（不可撤销，谨慎操作） |
| **重命名列** | 修改列标题 |

### 列 Markdown 结构

列在 Markdown 文件中以二级标题表示：

```markdown
## 待办

- [ ] 卡片 A
- [ ] 卡片 B

## 进行中

- [ ] 卡片 C
```

列的顺序即标题在文件中的顺序。拖拽列会改变标题在文件中的位置。
