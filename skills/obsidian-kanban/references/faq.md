# FAQ — 常见问题与已知问题

## Frontmatter 限制与注意事项

### 看板文件中的 Frontmatter

看板文件的 YAML frontmatter 中**只需**保留 `kanban-plugin` 声明：

```yaml
---
kanban-plugin: basic
---
```

### 常见问题

| 问题 | 说明 |
|------|------|
| 能否在 `kanban-plugin` 中使用其他值？ | 目前只有 `"basic"` 是官方支持的值。其他值无定义行为 |
| 能否在 frontmatter 中添加自定义字段？ | 可以，但看板插件不会读取或使用它们。这些字段仅供其他插件（如 Dataview）使用 |
| 删除 `kanban-plugin` 会怎样？ | 文件将不再被识别为看板，以普通 Markdown 渲染 |
| Frontmatter 中的自定义字段会影响看板吗？ | 不会。看板插件只读取 `kanban-plugin` 键 |

### 卡片中的 Frontmatter

- 从卡片创建的笔记可以包含完整的 frontmatter
- 通过 `metadata-keys` 设置，看板可以显示链接笔记 frontmatter 中的指定字段
- 卡片本身**不包含** frontmatter，其元数据以 `@{}`、`#tag` 等内联形式存储

## 已知限制

| 限制 | 说明 | 替代方案 |
|------|------|----------|
| **看板不能嵌入笔记** | `![[board.md]]` 不会渲染看板视图 | 看板必须作为独立文件打开 |
| **跨看板拖拽有限** | 部分版本支持，但可能不稳定 | 确保两个看板都在编辑模式下打开 |
| **移动端性能** | 大量卡片（500+）时移动端可能卡顿 | 使用归档功能控制卡片数量 |
| **卡片不支持多段落（早期版本）** | 旧版看板卡片仅支持单行 | 新版已支持多行，使用 Enter 或 Shift+Enter |
| **无原生甘特图** | 看板不提供时间线视图 | 结合其他插件或使用 Mermaid 甘特图 |
| **无原生筛选/过滤** | 看板不支持按标签/日期筛选显示 | 使用搜索功能或结合 Dataview |
| **无原生子任务** | 卡片内不支持嵌套子任务结构 | 使用 Tasks 插件的 `- [ ]` 嵌套列表 |
| **无自动化规则** | 不支持 "当卡片移动到X列时自动Y" | 使用第三方插件如 Kanban Status Updater |

## 故障排查

### 看板不显示

1. 检查 frontmatter 中是否有 `kanban-plugin: basic`
2. 确认插件已安装并启用
3. 尝试切换 Obsidian 的编辑/预览模式

### 卡片日期不显示

1. 检查 `hide-dates-in-title` 设置是否为 `true`
2. 确认日期格式为 `@{YYYY-MM-DD}`（注意大括号）
3. 检查日期触发字符是否被自定义修改

### 标签不显示颜色

1. 确认 `tag-colors` 配置中的标签名与实际标签完全匹配（包括大小写）
2. 标签颜色配置是 JSON 格式，确保语法正确

### 归档卡片丢失

1. 检查 `%% kanban:archive %%` 注释块是否被误删
2. 如果文件被外部编辑器修改，归档数据可能损坏
3. 建议定期备份看板文件
