# Obsidian Kanban 插件技能

在 Obsidian vault 中以 `.md` 文件形式创建和管理可视化看板。看板数据完全以 Markdown 存储，兼容 Obsidian 同步和版本控制。

**插件信息**：作者 mgmeyers · 4.1k stars · 最新版 2.0.51 · **正在寻找新维护者**
**官方文档**：https://publish.obsidian.md/kanban/
**核心格式**：Frontmatter 声明 + 二级标题列 + 列表项卡片

```markdown
---
kanban-plugin: basic
---

## 待办

- [ ] 任务一
- [ ] [[关联笔记]]

## 进行中

- [ ] 另一个任务 @{2026-04-01} #工作

## 已完成

- [x] 完成的任务 ✅
```

## 快速参考

| 你要做什么 | 该怎么做 |
|-----------|---------|
| 创建新看板 | `Ctrl+P` → `Kanban: Create new board`，或右键文件列表 → 新建看板 |
| 添加列/卡片 | 看板界面点击「+ 添加列」或列底部「+ 添加卡片」 |

| 关联笔记 | 卡片内输入 `[[` 弹出建议；或从文件面板**拖入**文件 |

| 设置精确时间 | 卡片内输入 `@@` 呼出时间选择器 |
| 标记完成 | 勾选复选框，或输入完成触发字符 `✅`（可自定义） |
| 从卡片新建笔记 | 右键卡片 → New note from card（自动双向链接，支持模板） |
| 归档已完成卡片 | 列菜单 → 归档已完成卡片；或命令面板 `Kanban: Archive completed cards` |
| 搜索卡片 | 看板右上角 🔍 按钮 |
| 切换视图 | 看板/表格/列表 三种视图，命令面板或看板头部按钮 |
| 跨看板拖拽 | 直接拖拽卡片到另一个看板窗口 |
| 配置看板选项 | 加载 `references/board-settings.md` |
| 结合 Tasks 插件 | 加载 `references/tasks-integration.md` |
| 获取配置模板 | 加载 `references/config-templates.md` |

## 关键规则

### ⚠️ 必须先读取文件

在创建、编辑或删除任何看板内容之前：

1. **读取** 目标 `.md` 文件，了解现有内容和结构
2. **识别** Frontmatter 区域（`---` 包裹）和看板数据区域
3. **理解** 列结构（`## 标题`）和卡片格式（`- [ ]` 或 `- [x]`）
4. **然后** 才进行修改，保持 Frontmatter、设置注释块（`%% kanban:settings %%`）和归档注释块（`%% kanban:archive %%`）不变

**避免：**
- ❌ 不读文件就直接编辑看板内容
- ❌ 破坏 Frontmatter 格式或 `%% kanban:settings %%` / `%% kanban:archive %%` 注释块
- ❌ 修改非目标列/卡片的内容

**遵循：**
- ✅ 读取文件 → 理解结构 → 精准编辑
- ✅ 新增卡片时匹配已有的卡片格式风格
- ✅ 保持 Frontmatter 中的 `kanban-plugin` 声明不变

### ⚠️ 看板与 Tasks 重复任务的限制

Kanban 内勾选复选框**不会**触发 Tasks 插件的重复任务逻辑。如需重复任务自动生成，须在 Tasks 查询视图或 Markdown 编辑器中完成勾选。详情见 `references/tasks-integration.md`。

### ⚠️ 看板不能嵌入普通笔记

Kanban 看板必须以独立 `.md` 文件存在，**不能**通过 `![[board.md]]` 嵌入到其他笔记中显示。这是当前的已知限制。

## 两阶段工作流

### 阶段一：创建/编辑看板

1. **读取** 目标文件，确认头部有 `---\nkanban-plugin: basic\n---`
2. **创建列**：添加 `## 列名` 二级标题（列之间用空行分隔）
3. **创建卡片**：在列标题下方添加 `- [ ] 卡片内容`
4. **设置日期**：卡片内加 `@{YYYY-MM-DD}` 格式日期
5. **设置时间**：卡片内加 `@@{HH:mm}` 格式时间
6. **添加标签**：卡片内加 `#标签名`
7. **关联笔记**：卡片内加 `[[笔记名]]`

### 阶段二：高级配置

1. **需要配置看板行为或外观时** → 加载 `references/board-settings.md`
2. **需要处理归档或列配置时** → 加载 `references/archive-and-lane.md`
3. **需要结合 Tasks 插件时** → 加载 `references/tasks-integration.md`
4. **需要快速搭建某类看板时** → 加载 `references/config-templates.md`
5. **需要了解 FAQ 和已知问题时** → 加载 `references/faq.md`

## 卡片内容支持

| 语法 | 说明 | 示例 |
|------|------|------|
| 复选框 | 勾选标记完成 | `- [ ]` / `- [x]` |
| 双向链接 | 关联笔记，`Ctrl+点击` 预览 | `[[笔记名]]` |
| 标签 | 卡片分类，可设置颜色和点击操作 | `#工作` `#紧急` |
| 日期 | 截止日期，超期变红，可链接每日笔记 | `@{2026-04-01}` |
| 时间 | 精确时间 | `@@{14:30}` |
| 格式化 | 加粗、高亮、代码等 | `**重要**`、`==关键==`、`` `code` `` |
| 图片 | 嵌入图片 | `![[图片.png]]` |
| 多行内容 | 卡片内换行继续书写（回车或 Shift+Enter） | 多行文本 |
| 外部链接 | 标准 Markdown 链接 | `[文字](https://...)` |

**Tasks 元数据（需安装 Tasks 插件）**：卡片中可使用 `📅` `🛫` `⏳` `⏫` `🔺` `🔽` `🔁` 等 Emoji，会被 Tasks 插件全局识别。

## 常见错误

| ❌ 错误写法 | ✅ 正确写法 | 原因 |
|------------|-----------|------|
| 删除 `%% kanban:settings %%` 注释块 | 保留该注释块不动 | 它存储了看板的全部配置 |
| 删除 `%% kanban:archive %%` 注释块 | 保留该注释块不动 | 它存储了已归档卡片数据 |
| 在 Frontmatter 中写列和卡片 | 列和卡片写在 `---` 之后 | Frontmatter 只用于 `kanban-plugin` 声明 |
| 列之间没有空行分隔 | 列标题之间保留一个空行 | 空行确保 Markdown 解析正确 |
| 日期写成 `@2026-04-01` | 日期写成 `@{2026-04-01}` | 大括号是标准格式 |
| 把 Tasks 的 `📅` 和看板的 `@{}` 混为一谈 | 两者可共存，但含义不同 | `@{}` 是看板日期，`📅` 是 Tasks 元数据 |
| 在看板里打勾期望 Tasks 自动重复 | 在 Tasks 查询视图里打勾 | 看板勾选不触发 Tasks 重复逻辑 |
| 尝试用 `![[board.md]]` 嵌入看板 | 看板必须作为独立文件打开 | 看板不支持嵌入显示 |
| 在 `kanban-plugin` 值上随意修改 | 保持 `kanban-plugin: basic` | 其他值无官方支持 |
